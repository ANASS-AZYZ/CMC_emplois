Cours Laravel Complet - De Scratch a Projet Grand Niveau

Objectif: Te faire passer du niveau debutant a la capacite de construire un projet Laravel solide, maintenable et deployable.

Auteur: GitHub Copilot | Version: 2026-03-05

PHP Laravel MySQL Blade API Auth Testing Deployment

0. Comment utiliser ce cours

  • Lis chaque module dans l ordre, sans sauter les bases.
  • A la fin de chaque module: fais un mini challenge.
  • Pratique 2 a 3 heures par jour minimum.
  • A la fin, construit un projet complet avec ton propre cahier des charges.

1. Prerequis indispensables

1.1 Prerequis techniques

  • PHP 8.2+
  • Composer
  • MySQL ou MariaDB
  • Node.js + npm (pour Vite/front)
  • VS Code

1.2 Bases a connaitre

  • PHP orientee objet (classes, methodes, namespaces)
  • SQL de base (CREATE, SELECT, JOIN, UPDATE)
  • HTTP (GET, POST, PUT, DELETE)
  • Git (commit, push, branch)

2. Installation Laravel de zero

composer create-project laravel/laravel mon-projet
cd mon-projet
cp .env.example .env
php artisan key:generate
php artisan serve

Configurer la DB dans .env:

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=ma_db
DB_USERNAME=root
DB_PASSWORD=
php artisan migrate

4. Routing + Controllers

// routes/web.php
Route::get('/products', [ProductController::class, 'index']);
Route::post('/products', [ProductController::class, 'store']);
php artisan make:controller ProductController
Regle d or: une route doit etre claire, un controller doit rester lisible.

5. Blade templating

  • Layouts: @extends, @section, @yield
  • Composants: <x-app-layout>
  • Boucles/conditions: @foreach, @if
  • Protection XSS par defaut avec {{ }}

6. Migrations, Models, Relations

6.1 Creer migration + model

php artisan make:model Post -m

6.2 Exemple relation

// User.php
public function posts() { return $this->hasMany(Post::class); }

// Post.php
public function user() { return $this->belongsTo(User::class); }

6.3 Bonnes pratiques

  • Noms de colonnes FK: user_id, groupe_id, etc.
  • Indexes sur colonnes recherchees
  • Migrations atomiques et reversibles

7. Validation des donnees

php artisan make:request StorePostRequest
public function rules(): array {
  return [
    'title' => ['required','string','max:255'],
    'content' => ['required','string'],
  ];
}

Utilise toujours des Form Requests dans les operations critiques.

8. Authentification + roles

  • Installer Breeze pour auth rapide
  • Ajouter un champ role dans users
  • Middleware role pour proteger routes admin/formateur
Route::middleware(['auth','role:admin'])->group(function () {
  Route::get('/admin/dashboard', ...);
});

9. CRUD propre (pattern complet)

  1. Route resource
  2. Controller resource
  3. Request validation
  4. Views index/create/edit/show
  5. Flash messages success/error
Route::resource('groupes', GroupeController::class);

10. Gestion des erreurs et logs

  • Utiliser try/catch seulement si necessaire
  • Laisser Laravel gerer les exceptions metier normales
  • Consulter storage/logs/laravel.log
  • Messages utilisateur simples, logs techniques detailles

11. Eloquent avance (important projet grand)

11.1 Eviter N+1

$seances = Seance::with(['groupe','salle','formateur'])->latest()->get();

11.2 Scopes

public function scopeActif($query) {
  return $query->where('active', true);
}

11.3 Transactions

DB::transaction(function () {
  // operations multiples critiques
});

12. API avec Laravel

  • Routes dans routes/api.php
  • Retour JSON propre et coherent
  • Resources API pour formatter les reponses
  • Auth API (Sanctum)

13. Files, mails, notifications

  • Upload fichiers: Storage
  • Emails: Mailables
  • Notifications: email, database, broadcast

14. Queues et jobs (projet grand)

Mettre les taches lourdes en arriere plan: emails, exports, traitements massifs.

php artisan queue:work

15. Caching et performances

  • Cache de requetes frequentes
  • Indexes SQL bien places
  • Pagination au lieu de charger tout
  • Profiling sur pages lentes

16. Testing (indispensable)

php artisan test
  • Feature tests: parcours utilisateur
  • Unit tests: logique metier pure
  • Tester auth, permissions, validations, conflits metier

17. Securite

  • CSRF actif sur formulaires
  • Validation stricte des inputs
  • Policies/Gates pour autorisations
  • Ne jamais exposer infos sensibles dans erreurs
  • Hasher mots de passe avec bcrypt (Hash::make)

18. Frontend dans Laravel

  • Blade + Vite pour SSR classique
  • Tailwind pour UI rapide
  • Possibilite SPA plus tard (Vue/React + API)

24. Deployment resume

  1. Mettre APP_ENV=production et APP_DEBUG=false
  2. Configurer DB et cache
  3. composer install --no-dev --optimize-autoloader
  4. php artisan migrate --force
  5. npm run build
  6. php artisan config:cache route:cache view:cache

25. Conseils pour devenir fort en Laravel

  • Code chaque jour, meme petit.
  • Lis du code propre open-source.
  • Ecris des tests des le debut.
  • Fais des revues de code.
  • Refactor regulierement.
